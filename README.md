This program serves as a tool to assist the ABCU advising department by loading a course list from a .csv file into an abstract data structure. Specifically, I implemented a binary search tree for efficient storage and retrieval. My approach began with analyzing and writing pseudocode for three common data structures and evaluating their Big-O runtime complexity to determine which would be most appropriate for this use case. Understanding these data structures is essential for evaluating the program’s performance, especially under worst-case conditions.

Because the end user requires fast searches and efficient memory usage, I chose to implement a binary search tree. Although a BST has a worst-case search complexity of O(n), a reasonably balanced tree provides an average time complexity of O(log n), which is significantly faster than a vector and generally more memory-efficient than a hash table.

This project expanded my approach to software design by pushing me outside my comfort zone, particularly in working with pointers and dynamically allocated nodes in C++. Prior to this course, I was hesitant to work with these concepts. However, after eight weeks of studying abstract data types, I now feel much more confident applying them in future projects. Additionally, the feedback from my professor helped me place greater emphasis on validation and meaningful inline commentary. Overall, this project has strengthened both my technical skills and my confidence as I move closer to a career in software development.
